## 功能设计
输入：
1.用户输入的自然语言要求；
2.用户上传的文档(PDF,WORD格式，不超过20MB，可不上传)；
3.用户选择的图片类型：共四种，机制通路图/实验流程图/技术路线图/科普插图
（并且针对每种类型的图片需要选择具体要求：图片风格/图片语言/信息密度/图片比例）
输出：
输出对应的医学绘图图片。输出结束后，用户可以直接保存，返回最初界面。

## 开发流程
1.小程序端使用获取微信聊天记录中的 PDF/Word 文件，上传至后端云存储，Text LLM将提取出的文本和图片类型要求发送文本大模型（Qwen-8B），配合特定的系统提示词进行信息浓缩。
2.提取用户意图，端传来的参数整合成 Qwen-Image 的Prompt，参数包括：
  - 图片类型 (Type)：机制通路图/实验流程图/技术路线图/科普插图，需要决定调用哪个专属的 LoRA 模型。
  - 图片风格 (Style)：映射为具体的 Prompt 标签，如 3D 风格 (3d render, octane render, volumetric lighting)、扁平化风格 (flat design, vector art, 2d minimalistic)。
  - 图片语言 (Language)：
  - 两个方案：1.Prompt 中强化文字渲染指令；2.通过 Prompt 控制生成带有空白文本框或标签底图的结构图，后续通过后端 OpenCV 或前端 Canvas 叠加文字
  - 信息密度 (Density)：映射提示词Prompt标签。高密度：highly detailed, complex network, intricate pathways；低密度：minimalist, clean background, simplified structure。
  - 图片比例 (Aspect Ratio)：Diffsynth 推理时需要填入参数，可以映射为 Diffsynth 推理时的宽高参数，如 16:9 (width=1024, height=576)，4:3 (width=1024, height=768)。
3.基于 Diffsynth + Qwen-Image 的模型训练
- 数据准备：
  - 收集高质量的医学图片数据集，分类为：机制通路图、实验流程图、技术路线图、科普插图。每类至少几百到上千张。
  - 使用模型（如Qwen-VL）对图片进行详细标注。标注必须包含风格、语言、信息密度、图片比例。
- Diffsynth 训练 (LoRA)：
  - 训练 4 个独立的 LoRA 权重文件
4.后端推理服务
使用 FastAPI封装Diffsynth 推理代码，基于FastAPI，Redis和Celery Work链路部署APP应用。
APP 把 PDF 和参数发给FastAPI。用户的消息打包到Redis 队列里排队，Celery Work进行分布式任务队列工作
